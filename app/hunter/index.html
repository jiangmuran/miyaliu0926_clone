凌晨三点，橘子再次在沙发腿上磨爪。不是出于恶意—— 是因为它的大脑已经连续 11小时 没有获得任何有效感知输入。 它的主人买过电动逗猫棒、买过自动激光、买过定时喂食器。 每一件都在 72小时内 被彻底无视。

习惯化（Habituation）是哺乳动物神经系统的基本特性： 可预测的刺激等于无刺激。 所有现有产品都是 静态程序，在猫的神经系统面前毫无胜算。

Hunter 是第一个能感知猫、理解猫、并在行为层面 持续进化 的自主系统。

02 — Core Insights

Five engineering theses

Hunter 的每一项设计决策都来自行为神经科学与工程约束的交叉点。

ε-greedy
惊喜熵引擎 (Surprise Entropy Engine)
基于 Shannon 熵与 ε-greedy Bandit 的行为调度器。实时量化猫的信息熵期望值，在 exploit（已知偏好）和 explore（新刺激）之间动态切换，从根本上对抗习惯化。
YOLOv8
实时猫咪感知 (Cat Perception)
YOLOv8-nano 在 Raspberry Pi 5 上实现 <80ms 推理延迟。结合 Lucas-Kanade 光流法计算活动度评分，PID 控制器实现亚厘米级距离跟随。
MFCC
非语言情感感知 (Vocal Emotion)
MFCC 特征提取 + CNN 分类器实时识别猫鸣声情绪（5类：hunger / content / alert / distress / purr）。声学情感状态注入行为决策图，驱动护理响应。
Beta-Bandit
个性化偏好记忆 (Memory Box)
Beta 分布 Bandit 模型持久化猫咪偏好权重。每次互动后贝叶斯更新，随时间收敛到个体最优策略。SQLite 持久化，支持跨设备迁移。
VLM+LLM
多模态日志系统 (Meme + Diary)
VLM 对猫咪高光时刻生成 meme 字幕，LLM 以猫咪第一视角撰写日记。Ring buffer 缓存最高 cuteness-score 帧，FCM 推送至主人 APP。
03 — Why Hunter Wins

Five theses no competitor can answer

每个卖点都对应一个现有产品无法解决的结构性问题，也对应评审的一个评分维度。

01
创新性
完整的生物学闭环 — 现有 1000+ SKU，无一做到
猫的捕猎行为在神经科学层面分三阶段：寻找 → 追逐 → 捕获。任何一个阶段缺失，多巴胺回路都不完整。这就是为什么激光玩具会让猫焦虑——追逐回路启动，捕获回路永远不关闭，长期引发强迫症行为。

Hunter 构建了完整的狩猎 → 捕获 → 进食闭环：机械臂模拟猎物动态，猫扑中时底盘自动投出零食作为奖励。市面上 SKU 超过 1000 款的自动逗猫产品，没有一款实现了这个闭环。这是空白。
02
技术难度
AI 必须主动制造意外 — 惊喜熵引擎
所有 AI 产品都在优化：追求"最准确"、"最点击"。但优化是猫无聊的根源。

Hunter 引入全新指标：惊喜熵（Surprise Entropy）——实时衡量当前动作对这只猫的不可预测程度。算法目标不是"让猫扑得更准"，而是维持70% 把握 + 30% 不确定的心流区间（Csikszentmihalyi 心流理论的工程实现）。

Hunter 会故意失败（让猫扑中）、故意打破规律（在猫预测下一步前反转）、故意"笨拙"（模拟受伤猎物）。相当于给猫配了一个 AlphaGo——但方向相反：目标不是赢，而是永远恰好比你强一点点。
03
商业潜力
第一个为"不会说话的存在"设计的 AI
现有所有 AI 产品——ChatGPT、Siri、Copilot——都依赖语言作为接口。Hunter 读的是尾巴姿态、耳朵角度、全身体态和声纹情绪。这是 Ambient Computing 的极致：AI 在没有任何主动指令的情况下读懂主体。

英文定义："AI for the Inarticulate." 今天做猫，明天技术栈可平移到：失能老人（中国 60岁以上 2.8亿）、低龄婴儿、特殊需求儿童。猫是起点，不是终点。
04
护城河
每台 Hunter 最终都不一样 — 真正的 AI 原生硬件
传统硬件：出厂是什么样，用一年还是什么样。Hunter 出厂时相同，但和"它的那只猫"持续共同进化：

这只猫不喜欢被快速逼近 → Hunter 永久记录，调低逼近速度。这只猫偏好激光胜过逗猫棒 → 权重倾斜，但偶尔切换保持新鲜感。这只猫每天 17:30 最活跃 → 自动在这个时段加大强度。

六个月后，张三家的 Hunter 和李四家的 Hunter，"灵魂"已经不一样了。用户换设备 = 失去六个月的人格化记忆。Churn 率接近零，时间站在我们这边。
05
情感穿透力
AI 日记和记忆盒子 — 猫的整个生命
每晚推送的不是数据图表，是 LLM 写的文字：

"今天 Mochi 在阳光下睡了两小时，醒来后试图捕捉那只羽毛玩具 23 次，差两次几乎得手，失手后假装根本不在乎。下午四点，它坐在窗台上盯着楼下的鸽子看了整整 40 分钟，尾巴的节奏是我见过最专注的样子。"
Hunter 在猫的整个生命中持续记录每一个互动瞬间。猫离世后，自动整理「它的一生」。

调研：73% 宠物主最担忧猫独处时是否孤独，68% 认为无法记录猫的日常是最大遗憾。Hunter 同时解决这两个问题。营销只需一句话："Hunter 会记得它生命的每一天。"

Main Loop Pseudocode
# Hunter main control loop — 10Hz
LOOP main_loop(10 Hz):
    frame   ← camera.capture()
    cats    ← yolo.detect(frame)           # YOLOv8-nano, <80ms
    audio   ← mic.sample(0.1s)
    emotion ← sound_classifier.infer(audio) # shared state

    IF cats.found:
        activity ← optical_flow.score(frame, prev_frame)
        distance ← cats[0].bbox.depth_estimate()
        pid.update(distance)                 # chase / maintain

        action ← entropy_engine.select(cats[0], emotion)
        dispatch(action)                      # hunt | care | idle

        IF action.reward_observed:
            bandit.update(action.arm, reward)  # Beta posterior

    health_monitor.tick()
    meme_generator.tick(frame, cats)
    daily_diary.accumulate(cats, emotion)

    prev_frame ← frame

11 — Risk Register

Known risks, prepared responses

风险概率应对方案
YOLO 检测延迟影响体验 中 赛前测试推理速度，用 YOLOv8 nano；备用：纯运动检测降级模式（无需模型，<5ms）

投食机构现场卡顿 低 演示中用手动投食替代，聚焦算法展示；硬件问题不影响 AI 部分评分

YOLO 推理延迟 >150ms 中 预加载模型到内存；开启 TFLite INT8 量化；最差情况降帧到 15fps 保住跟随效果
